Application Definition Statement

This application will find a magic card, searched by the user, and display data and possible places to buy the card. This will be intended for anyone looking to buy magic cards or for people who just want to look them up.

Web services

  1. MTG API: DeckBrew API
  2. Ebay API Ebay Dev. center

Features

  1. search for any magic card
  2. gives a link to an ebay search
  3. ability to filter searches for more accuracy
  4. multiple pages if searches return more than 100 results

Design Decisions

Our passion for magic led to the decision to take the magic api and the ebay api and combine them. We wanted to create a place where people could search for the cards they want, or even list all of the cards to look through, and be given a place that they could buy it from. We thought that a very simple and clean design would be to have a filter block above the search bar.

Technical Decisions

I wanted to be able to strictly filter using the colors of the cards rather than just returning either one color or the other, but it turned out to be more complicated than I thought and I also did not have enough time to experiment with it. I really wanted to add multiple pages so I added a button if there are 100 results, because only 100 can be returned, and if someone clicks on it the site will display the next page of results.

Extras

The extras are the filtered search system and the page system.
